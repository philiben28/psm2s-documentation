# Bienvenue dans le projet PSM2S

Vous intervenez sur un projet Django existant.

Avant toute modification :

1. Lire IA_RULES.md
2. Lire DEVELOPMENT_PROTOCOL.md
3. Lire VISION_PRODUIT.md
4. Lire PRODUCT_BACKLOG.md
5. Lire DECISIONS.md

Aucune modification du code n'est autorisée avant cette lecture.

Vous devez ensuite analyser le projet existant.

Vous devez proposer un plan de développement.

Vous devez attendre la validation du Product Owner.

Vous ne devez jamais développer plusieurs fonctionnalités importantes simultanément.

Le calendrier réglementaire est actuellement la priorité stratégique du projet.
